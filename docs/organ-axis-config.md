# 장기별 차트 축 자동 전환 기능

## 한 줄 요약

사용자가 바디맵에서 장기를 클릭하면, 그 장기 데이터에 맞는 차트 축이 자동으로 바뀐다.

---

## 왜 필요한가?

단일 세포 데이터는 장기마다 분석 방법이 다르다.

| 장기 | 차원 축소 방법 | 클러스터링 기준 |
|------|--------------|----------------|
| 폐 | UMAP | 세포 유형 (cell_type) |
| 심장 | PCA | Leiden 클러스터 |
| 뇌 | tSNE | 뇌 영역 (brain_region) |
| 신장 | UMAP | 네프론 구역 (nephron_segment) |

모든 장기에 동일한 축을 쓰면 데이터가 의미 없이 뭉개진다.  
**장기마다 "정답 축"이 따로 있다.**

---

## 사용자 경험 흐름

```
1. 바디맵에서 "심장" 클릭
        ↓
2. 시스템이 심장의 축 설정을 자동으로 가져옴
   → X축: X_pca_1  /  Y축: X_pca_2  /  색상: leiden_cluster
        ↓
3. UMAP 차트가 심장 PCA 뷰로 자동 전환
        ↓
4. "폐"를 추가로 클릭하면?
   → 폐(UMAP)와 심장(PCA)이 다름 → 첫 번째 선택(심장) 기준으로 표시
   → UI에 "primary" 배지로 안내
```

사용자는 축 이름을 알 필요가 없다.  
그냥 클릭하면 맞는 뷰가 나온다.

---

## 내부 동작 구조

### 축 결정 로직 (`resolveAxisConfig`)

| 상황 | 결과 | 배지 |
|------|------|------|
| 아무 장기도 선택 안 함 | 기본값 사용 | `default` |
| 하나만 선택, 또는 여러 개인데 축이 동일 | 해당 장기 축 사용 | `organ` |
| 여러 장기 선택, 축이 서로 다름 | 첫 번째 선택 장기 기준 | `primary` |

### React 훅 (`useOrganAxisConfig`)

컴포넌트 어디서든 한 줄로 가져다 쓸 수 있다.

```ts
const axisConfig = useOrganAxisConfig(selectedOrgans);
// axisConfig.xAxis, axisConfig.yAxis, axisConfig.colorBy
// → 선택된 장기가 바뀔 때마다 자동으로 업데이트됨
```

차트 컴포넌트는 이 값을 그냥 받아서 쓰면 된다.  
"어떤 장기가 선택됐는지"는 신경 쓸 필요 없다.

---

## 축 설정 관리 방식

### 현재: JSON 파일 (`organ-axis-config.json`)

축 설정은 코드와 분리된 JSON 파일 하나로 관리한다.

```json
{
  "heart": {
    "xAxis":   "X_pca_1",
    "yAxis":   "X_pca_2",
    "colorBy": "leiden_cluster",
    "label":   "Heart — cardiac cluster PCA"
  },
  "lung_right": {
    "xAxis":   "X_umap_1",
    "yAxis":   "X_umap_2",
    "colorBy": "cell_type",
    "label":   "Lung — cell type UMAP"
  }
}
```

**장점**
- 연구자가 코드 없이 직접 편집 가능
- 새 장기 추가 = JSON에 항목 하나 추가, 코드 수정 없음
- 나중에 서버 API에서 불러오는 구조로 자연스럽게 확장 가능

**단점**
- 컬럼 이름 오타 등을 빌드 시점에 잡지 못함 (런타임에서만 확인됨)

---

### 미래: 데이터셋 메타데이터에서 자동 파생

각 장기 데이터셋(`.h5ad` 파일 등)이 자체적으로 "이 데이터는 UMAP으로 만들어졌고, cell_type으로 색칠하라"는 메타데이터를 포함하게 되면, JSON 설정표 없이도 축이 자동으로 결정된다.

```
데이터셋 로드
    ↓
파일 내 메타데이터 읽기
    { embedding: "umap", colorBy: "cell_type", ... }
    ↓
축 자동 결정 — 별도 설정 불필요
```

**장점**
- 새 장기 데이터를 추가해도 설정 파일 수정이 필요 없음
- 데이터와 설정이 항상 일치 (불일치 오류 원천 차단)

**단점**
- 데이터셋 포맷 표준화가 선행되어야 함
- 현재 데이터 파이프라인이 이 메타데이터를 지원하지 않음

> 데이터 파이프라인이 성숙되면 JSON 방식에서 이 방식으로 전환한다.  
> 전환 시 `resolveAxisConfig` 함수의 내부만 바꾸면 되고, 나머지 코드(훅, 차트)는 그대로다.

---

## 현재 구현 상태

| 항목 | 상태 |
|------|------|
| JSON 축 설정 파일 | ✅ 완료 (placeholder 값) |
| 축 결정 로직 | ✅ 완료 |
| React 훅 | ✅ 완료 |
| 바디맵 → 훅 연결 | ✅ 완료 |
| 실제 차트에 축 적용 | ⏳ 데이터셋 온보딩 후 진행 예정 |
| 실제 축 이름 확정 | ⏳ 도메인 전문가 확인 후 진행 예정 |

> 데이터가 준비되면 `organ-axis-config.json`의 값만 채워 넣으면 전체 기능이 즉시 동작한다.
